# 视频内容分类与动效规则

本文档定义 FrameCraft Agent 流程中的基础内容分类。Agent 在 `content_category=auto` 时会结合 `target_style`、素材备注、ASR 文本自动判断。

## 1. 搞笑整活 `funny_reaction`

- 适用：鬼畜、搞笑吐槽、梗视频、强反应类口播。
- 画面策略：优先保留人物表情与反应节奏，字幕与动效偏夸张、节拍感更强，允许“小贴纸 + 小梗字 + 单个大反应”分层共存。
- 推荐元素：
  - 关键词弹出 `keyword_pop`
  - 反应气泡 `reaction_bubble`
  - 漫画标题条 `comic_caption`
  - 梗图贴纸 `meme_sticker`
  - 爆点星芒 `impact_flash`
  - 表情爆点 `emoji_burst`
- Agent 设计要求：
  - 允许更密集的表情/贴纸/反应卡，但必须像综艺包装一样分层：一个主反应、一个辅助梗、一个背景节奏元素。
  - 表情动画需要持续抖动、呼吸、弹跳或漂移，不能只是静态贴图。
  - 梗文案必须面向观众，使用简体中文短句，避免解释制作意图。
- 约束：
  - 表情动画只能落在安全区，不得压到脸部。
  - 优先使用现成贴纸/表情资产，不手搓高风险复杂骨骼动画。
  - 同一时段允许“一个主强调 + 一个轻贴纸”，但不得满屏打架。

## 2. 知识科普 `knowledge_explainer`

- 适用：知识讲解、教程、分析拆解、方法论视频。
- 画面策略：强调结构、重点、流程化解释、对照表与数字说明。
- 推荐元素：
  - 章节卡 `chapter_card`
  - 信息表 `info_table`
  - 对照表 `compare_table`
  - 流程图 `process_flow`
  - 数据块 `stat_block`
  - 解释卡 `explainer_card`
- Agent 设计要求：
  - 表格、流程图、数据卡优先，装饰性贴纸降级。
  - 流程必须逐步出现；每一步出现后要给观众至少 1 秒阅读时间。
  - 若引用外部数据、机构名称、统计口径，左下角用小字标明出处。
- 约束：
  - 表格/数据块优先放在非人物区域。
  - 流程图优先使用 3 步以内的短链路，保证一眼能懂。
  - 若出现外部数据引用，左下角必须附小字来源。

## 3. 产品演示 `product_demo`

- 适用：产品介绍、SaaS demo、功能说明、网页/录屏类视频。
- 画面策略：人物表达与界面说明并重，突出“看哪里、做什么、为什么有用”，并适度加入功能网格和流程步骤。
- 推荐元素：
  - 解释卡 `explainer_card`
  - 圈选标注 `annotation`
  - 功能网格 `feature_grid`
  - 操作流程 `process_flow`
  - 对照表 `compare_table`
  - 数据块 `stat_block`
  - CTA `cta`
- 约束：
  - 标注圈选必须与目标界面元素对齐。
  - 若人物不全屏，界面卡片优先放内容区，避免压住面部。

## 4. 观点评论 `opinion_commentary`

- 适用：评论、观点表达、人物口播、时事/泛话题输出。
- 画面策略：突出观点推进、金句记忆点、论证结构和节奏层次。
- 推荐元素：
  - 关键词弹出 `keyword_pop`
  - 金句框 `quote_panel`
  - 章节卡 `chapter_card`
  - 论点卡 `thesis_card`
  - 对照表 `compare_table`
  - 逻辑流程 `process_flow`
  - 数据块 `stat_block`
- Agent 设计要求：
  - 把观点拆成“变量、对照、验证、结论”等结构块，而不是重复使用同一种卡片。
  - 流程图逐节点出现；对照表逐行出现；金句卡只用于真正的记忆点。
  - 非全屏横版要充分利用左侧信息区，安全前提下避免大面积空白。
- 约束：
  - 文案必须面向观众，不得出现制作侧词汇。
  - 字幕始终逐字，不允许总结式改写字幕。

## 5. 高级口播密度规则

- 结构型图形：`thesis_card` / `compare_table` / `process_flow` / `feature_grid` 负责“高级感”，默认优先级高于纯装饰。
- 强调型图形：`keyword_pop` / `stat_block` / `impact_flash` 负责节奏点，不应连续霸屏。
- 趣味型图形：`reaction_bubble` / `comic_caption` / `meme_sticker` 仅用于搞笑整活，且必须走人物安全区。
- 画面避让：允许“大卡片 + 小强调”同屏，但同一时段最多一个结构型图形。
- 字体语言：所有字幕和叠加文案统一简体中文；搞笑类允许语气夸张，知识/观点/产品类优先克制、清晰、像编辑设计而非贴纸堆砌。
- 句式规则：叠加文案尽量不要使用“不是……而是……”包装句，优先用直接判断、动词短句和名词结构。
