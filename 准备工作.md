## 3. 定位剪映 / CapCut 安装位置

### 3.1 剪映专业版（本机实测）

| 项目 | 路径 |
|------|------|
| 程序本体 | `F:\JianyingPro\JianyingPro.exe` |
| 用户数据目录 | `C:\Users\ZHOU\AppData\Local\JianyingPro` |
| **草稿导入目录** | `C:\Users\ZHOU\AppData\Local\JianyingPro\User Data\Projects\com.lveditor.draft` |

> 草稿目录是 VectCutAPI 生成 `dfd_*` 文件夹后的复制目标。复制后需**重启剪映**才能在列表中看到。

### 3.2 手动查找方法

**方法一：开始菜单快捷方式**

```powershell
$sh = New-Object -ComObject WScript.Shell
$lnk = $sh.CreateShortcut("$env:APPDATA\Microsoft\Windows\Start Menu\Programs\剪映专业版\剪映专业版.lnk")
$lnk.TargetPath          # 程序 exe
(Get-Item $lnk.TargetPath).VersionInfo.FileVersion  # 版本号
```

**方法二：常见默认路径**

```text
# 剪映（中文）
%LOCALAPPDATA%\JianyingPro
草稿: %LOCALAPPDATA%\JianyingPro\User Data\Projects\com.lveditor.draft

# CapCut 国际版
%LOCALAPPDATA%\CapCut
草稿: %LOCALAPPDATA%\CapCut\User Data\Projects\com.lveditor.draft
```

**方法三：注册表（可选）**

```powershell
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*, `
  HKLM:\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall\* `
  -ErrorAction SilentlyContinue |
  Where-Object { $_.DisplayName -match '剪映|Jianying|CapCut' } |
  Select-Object DisplayName, InstallLocation, DisplayVersion
```

### 3.3 写入项目配置

将实际路径填入 `config/local.paths.json`（换机后必改）：

```json
{
  "jianying": {
    "app_exe": "F:\\JianyingPro\\JianyingPro.exe",
    "app_version": "9.7.1.13726",
    "data_dir": "C:\\Users\\ZHOU\\AppData\\Local\\JianyingPro",
    "draft_dir": "C:\\Users\\ZHOU\\AppData\\Local\\JianyingPro\\User Data\\Projects\\com.lveditor.draft",
    "is_capcut_env": false
  }
}
```

- 使用 **CapCut 国际版** 时，将 `is_capcut_env` 改为 `true`，并更新 `draft_dir` 为 CapCut 对应路径。

## 4. 安装基础工具

### 4.1 Node.js（必须 ≥ 22.19）

OpenClaw 要求 Node **22.19+**，HyperFrames 要求 Node **22+**。

```powershell
# 检查版本
node --version
npm --version
```

**安装方式（任选其一）：**

```powershell
# 方式 A：winget
winget install OpenJS.NodeJS

# 方式 B：NVM for Windows（推荐，便于切换版本）
winget install CoreyButler.NVMforWindows
# 安装后新开终端：
nvm install 22.21.1
nvm use 22.21.1
```

**⚠️ PATH 陷阱：** 若 Python 的 `Scripts` 目录下有同名 `node.exe`，会遮蔽真正的 Node。  
确保以下路径排在 Python Scripts **之前**：

```text
C:\Users\<你>\AppData\Local\Programs\node-v22.20.0-win-x64
C:\Users\<你>\AppData\Roaming\npm
```

验证：

```powershell
where.exe node
# 应指向 node-v22.x 或 Program Files\nodejs，而非 Python312\Scripts\node.exe
```

### 4.2 FFmpeg

```powershell
ffmpeg -version
```

未安装时可用 MSYS2、winget 或从 [ffmpeg.org](https://ffmpeg.org/download.html) 下载，并加入 PATH。

### 4.3 Python 3.10+

```powershell
python --version
pip --version
```

### 4.4 Git

```powershell
git --version
```

### 5.2 预下载 Chromium（可选，加速首次渲染）

HyperFrames 渲染依赖 Chromium。首次 `render` 会自动下载；也可提前安装：


下载后浏览器位于项目内 `chrome/` 目录。渲染时可指定：

```powershell
npx hyperframes render --browser-path "<项目根>/chrome/win64-xxx/chrome-win64/chrome.exe"
```

## 7. 安装 VectCutAPI（剪映草稿导出）

### 7.1 克隆与虚拟环境

```powershell
cd <项目根目录>
mkdir vendor -ErrorAction SilentlyContinue

git clone --depth 1 https://github.com/sun-guannan/VectCutAPI.git vendor/VectCutAPI
cd vendor/VectCutAPI

python -m venv venv-capcut
.\venv-capcut\Scripts\activate
pip install -r requirements.txt
```

### 7.2 配置文件

复制并编辑 `config.json`（中文剪映示例）：

```json
{
  "is_capcut_env": false,
  "draft_domain": "https://www.capcutapi.top",
  "port": 9001,
  "preview_router": "/draft/downloader",
  "is_upload_draft": false
}
```

- `is_capcut_env: false` → 中文剪映  
- `is_capcut_env: true` → CapCut 国际版

### 7.3 启动服务

```powershell
cd vendor/VectCutAPI
.\venv-capcut\Scripts\activate
python capcut_server.py
# 默认监听 http://127.0.0.1:9001
```

服务启动后，根路径可能返回 404，属正常；实际使用 `/create_draft`、`/add_video`、`/save_draft` 等接口。

### 7.4 草稿导入剪映流程

1. 调用 `save_draft` 后，在 `capcut_server.py` 运行目录下生成 `dfd_*` 文件夹  
2. 将整个 `dfd_*` 文件夹复制到剪映草稿目录（见第 3 节）  
3. 重启剪映专业版  
4. 在草稿列表中打开，验证视频轨、字幕、图片是否可编辑

参考：[VectCutAPI README](https://github.com/sun-guannan/VectCutAPI)

## 11. 各工具在项目中的角色

```text
用户素材 + 备注
        ↓
  OpenClaw Agent（剪辑规划、对话修改）
        ↓
  ASR / VLM（口播转录、画面理解）
        ↓
  unified_timeline.json（唯一事实来源）
        ↓
  ├── HyperFrames → preview.mp4（高级预览）
  └── VectCutAPI  → draft.zip（剪映可编辑草稿）
```

| 工具 | 职责 |
|------|------|
| **OpenClaw** | 调用 LLM、生成 `edit_plan.json` 和 timeline patch、调度工具 |
| **HyperFrames** | HTML 合成 → 无头 Chrome 截帧 → FFmpeg 输出 MP4 |
| **VectCutAPI** | 将时间线元素写入剪映草稿格式（视频/字幕/图片/音频/关键帧） |
| **FFmpeg** | 抽音频、转码、混音 |
| **Chromium** | HyperFrames 渲染引擎的浏览器内核 |
| **faster-whisper** | 本地口播 ASR，输出逐词时间戳 |

## 13. 常见问题

### Q1：`node --version` 报错 `No module named 'nodejs_wheel'`

Python Scripts 里的假 `node.exe` 抢了 PATH。把真实 Node 目录放到 PATH 最前面（见 4.1 节）。

### Q2：OpenClaw 报 Node 版本不足

当前 Node 为 22.14 等低于 22.19 的版本。升级到 22.19+ 后重新运行 `openclaw --version`。

### Q3：剪映里看不到导入的草稿

1. 确认复制到 `com.lveditor.draft` 目录（不是 `Projects` 上一级）  
2. 文件夹名以 `dfd_` 开头  
3. 重启剪映  
4. 确认 `config.json` 中 `is_capcut_env` 与所用软件一致

### Q4：HyperFrames 首次渲染很慢

正常现象。可先输出 720p 预览，或提前下载 Chromium（见 5.2 节）。

### Q5：VectCutAPI 与 HyperFrames 效果不一致

复杂 CSS/GSAP 动效无法完全转为剪映原生元素，需烘焙为视频覆盖层。见需求文档 4.2、9.5 节。
