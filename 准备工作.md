# 帧造 Agent 开发环境准备工作

> 本文档记录 Windows 本机从零搭建「帧造 Agent」所需依赖的步骤，便于换机或团队成员复现。  
> 对应需求文档：`帧造_Agent_项目需求文档(2).md`

---

## 1. 准备工作目标

在正式开发前，需要确认并安装以下能力：

| 能力 | 用途 |
|------|------|
| 剪映 / CapCut 路径定位 | 草稿导入验证 |
| Node.js 22.19+ | HyperFrames、OpenClaw |
| FFmpeg | 音视频处理、HyperFrames 编码 |
| Python 3.10+ | VectCutAPI、本地 ASR |
| HyperFrames | 高级预览视频渲染 |
| Chromium | HyperFrames 无头浏览器渲染 |
| VectCutAPI | 剪映 / CapCut 草稿导出 |
| OpenClaw | Agent 编排层 |
| ASR（faster-whisper） | 口播转录（**推荐/默认本地**） |
| 大模型 API Key | OpenClaw 分析 / 生成 / 对话（**必须**） |

## 2.2 换机推荐安装顺序（必读）

按以下顺序操作，可避开常见踩坑（ASR 缺失、剪映导出 `oss2` 报错、OpenClaw 网关凭证、前端 rolldown 等）：

```powershell
# 1. 克隆
git clone https://github.com/Karl-XZ/FrameCraft-Agent.git
cd FrameCraft-Agent

# 2. 安装 OpenClaw（全局，需 Node >= 22.19）
npm install -g openclaw@latest
openclaw --version    # 期望 OpenClaw 2026.6.x

# 3. 安装 HyperFrames + 前端依赖（必须用 Node 22，见 4.1 节 PATH 陷阱）
npm install
cd framecraft-agent && npm install && cd ..

# 4. 修改本机路径（剪映草稿目录、node_exe 等）
#    编辑 config/local.paths.json

# 5. 一键安装 Python 运行时依赖（后端 venv、ASR、VectCutAPI、OpenClaw 网关）
powershell -ExecutionPolicy Bypass -File scripts\setup-deps.ps1

# 6. 环境自检
powershell -ExecutionPolicy Bypass -File scripts\verify-env.ps1

# 7. 配置大模型 API Key（必须，否则分析/生成/对话均无法启动）
#    Web 工作台「模型设置」，或环境变量 LLM_API_KEY / LLM_BASE_URL

# 8. 启动（两个终端）
scripts\start-backend.bat    # 启动前会自动再跑 setup-deps
scripts\start-frontend.bat

# 9. 打开工作台
# http://127.0.0.1:5173/studio
```

确认 `http://127.0.0.1:8000/api/health` 中 `"openclaw": true`。

### 2.3 已自动化 vs 仍需手动

| 项目 | 是否自动 | 说明 |
|------|----------|------|
| 后端 Python venv + `requirements.txt` | ✅ `setup-deps.ps1` | `start-backend.bat` 启动前也会执行 |
| VectCutAPI 依赖（`oss2`、`flask` 等） | ✅ 同上 | 装入 **backend venv**（草稿导出走该解释器） |
| ASR `faster-whisper`（`vendor/asr-venv`） | ✅ 同上 | 无此环境会提示「口播转录失败，成片将无逐词字幕」 |
| OpenClaw 网关 `gateway.auth.mode=none` | ✅ 同上 + 后端启动时 patch | 避免对话报 `GatewayCredentialsRequiredError` |
| 对话任务走 `--local` 模式 | ✅ 代码内置 | 不依赖 Gateway WebSocket 凭证 |
| Node.js **≥ 22.19** 且在 PATH 最前 | ❌ 手动 | v20 会导致 HyperFrames / Vite rolldown 原生模块失败 |
| `npm install -g openclaw` | ❌ 手动 | 未安装时 setup 仅 WARN，不会自动装 |
| 根目录 + `framecraft-agent` 的 `npm install` | ❌ 手动 | 尚未并入 setup-deps |
| `config/local.paths.json` 本机路径 | ❌ 手动 | 剪映草稿目录、`node_exe` 换机必改 |
| 大模型 API Key | ❌ 手动 | DashScope / OpenAI 等，勿提交 Git |
| FFmpeg、Python、Git | ❌ 手动 | 系统级工具需事先安装 |

> **结论：** 按 2.2 节顺序安装并跑通 `verify-env.ps1` 后，**今天踩过的后端/ASR/剪映导出/OpenClaw 对话类问题不应再出现**；Node 版本、前端 npm、API Key 仍需每台机器自行确认。

## 3. 定位剪映 / CapCut 安装位置

### 3.1 剪映专业版（本机实测）

| 项目 | 路径 |
|------|------|
| 程序本体 | `F:\JianyingPro\JianyingPro.exe` |
| 用户数据目录 | `C:\Users\ZHOU\AppData\Local\JianyingPro` |
| **草稿导入目录** | `C:\Users\ZHOU\AppData\Local\JianyingPro\User Data\Projects\com.lveditor.draft` |

> 草稿目录是 VectCutAPI 生成 `dfd_*` 文件夹后的复制目标。复制后需**重启剪映**才能在列表中看到。

### 3.2 手动查找方法

**方法一：开始菜单快捷方式**

```powershell
$sh = New-Object -ComObject WScript.Shell
$lnk = $sh.CreateShortcut("$env:APPDATA\Microsoft\Windows\Start Menu\Programs\剪映专业版\剪映专业版.lnk")
$lnk.TargetPath          # 程序 exe
(Get-Item $lnk.TargetPath).VersionInfo.FileVersion  # 版本号
```

**方法二：常见默认路径**

```text
# 剪映（中文）
%LOCALAPPDATA%\JianyingPro
草稿: %LOCALAPPDATA%\JianyingPro\User Data\Projects\com.lveditor.draft

# CapCut 国际版
%LOCALAPPDATA%\CapCut
草稿: %LOCALAPPDATA%\CapCut\User Data\Projects\com.lveditor.draft
```

**方法三：注册表（可选）**

```powershell
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*, `
  HKLM:\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall\* `
  -ErrorAction SilentlyContinue |
  Where-Object { $_.DisplayName -match '剪映|Jianying|CapCut' } |
  Select-Object DisplayName, InstallLocation, DisplayVersion
```

### 3.3 写入项目配置

将实际路径填入 `config/local.paths.json`（换机后必改）：

```json

    "draft_dir": "C:\\Users\\<你的用户名>\\AppData\\Local\\JianyingPro\\User Data\\Projects\\com.lveditor.draft",

```

- 使用 **CapCut 国际版** 时，将 `is_capcut_env` 改为 `true`，并更新 `draft_dir` 为 CapCut 对应路径。

## 4. 安装基础工具

### 4.1 Node.js（必须 ≥ 22.19）

OpenClaw 要求 Node **22.19+**，HyperFrames 要求 Node **22+**。

```powershell
# 检查版本
node --version
npm --version
```

**安装方式（任选其一）：**

```powershell
# 方式 A：winget
winget install OpenJS.NodeJS

# 方式 B：NVM for Windows（推荐，便于切换版本）
winget install CoreyButler.NVMforWindows
# 安装后新开终端：
nvm install 22.21.1
nvm use 22.21.1
```

**⚠️ PATH 陷阱：** 若 Python 的 `Scripts` 目录下有同名 `node.exe`，会遮蔽真正的 Node。  
确保以下路径排在 Python Scripts **之前**：

```text
C:\Users\<你>\AppData\Local\Programs\node-v22.20.0-win-x64
C:\Users\<你>\AppData\Roaming\npm
```

验证：

```powershell
where.exe node
node --version   # 必须 >= v22.19.0
# 应指向 node-v22.x 或 Program Files\nodejs，而非 Python312\Scripts\node.exe
```

**前端 rolldown 报错时：** 若 `npm run dev` 提示 `Cannot find module '@rolldown/binding-win32-x64-msvc'`，说明用了 Node 20 安装依赖。处理：

```powershell
# 确保 Node 22 在 PATH 最前，然后重装前端依赖
cd framecraft-agent
Remove-Item -Recurse -Force node_modules -ErrorAction SilentlyContinue
Remove-Item -Force package-lock.json -ErrorAction SilentlyContinue
npm install
```


未安装时可用 MSYS2、winget 或从 [ffmpeg.org](https://ffmpeg.org/download.html) 下载，并加入 PATH。

### 4.3 Python 3.10+

```powershell

```

### 4.4 Git


**常见问题：**

| 现象 | 处理 |
|------|------|
| `Node.js v22.19+ is required` | 升级 Node 到 22.19+ |
| `openclaw: command not found` | 将 `%APPDATA%\npm` 加入 PATH |

### 6.1 网关凭证（本地开发自动配置）

FrameCraft 在本机以 **loopback 免凭证** 模式运行 Gateway。`scripts/setup-deps.ps1` 会向 `~/.openclaw/openclaw.json` 写入：

```json
{ "gateway": { "auth": { "mode": "none" } } }
```

后端启动时也会再次 patch。若对话仍报 `GatewayCredentialsRequiredError`，请重新执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup-deps.ps1
```

## 7. VectCutAPI（剪映草稿导出）

仓库已自带 `vendor/VectCutAPI/`（克隆项目即可，无需再 `git clone`）。

### 7.1 依赖安装（推荐：一键脚本）

**默认方式：** `setup-deps.ps1` 将 `vendor/VectCutAPI/requirements.txt`（含 `oss2`、`flask` 等）安装进 **backend venv**。  
未安装时生成阶段可能在 88%「注册版本与附件」失败，报错 `No module named 'oss2'`。

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup-deps.ps1
backend\venv\Scripts\python.exe -c "import oss2, flask; print('OK')"
```

### 7.2 独立 venv（可选，仅调试 capcut_server 时用）

```powershell
cd vendor/VectCutAPI
python -m venv venv-capcut
.\venv-capcut\Scripts\activate
pip install -r requirements.txt
```

### 7.3 配置文件

复制并编辑 `config.json`（中文剪映示例）：

```json
{
  "is_capcut_env": false,
  "draft_domain": "https://www.capcutapi.top",
  "port": 9001,
  "preview_router": "/draft/downloader",
  "is_upload_draft": false
}
```

- `is_capcut_env: false` → 中文剪映  
- `is_capcut_env: true` → CapCut 国际版

### 7.4 启动独立服务（可选）

```powershell
cd vendor/VectCutAPI
.\venv-capcut\Scripts\activate
python capcut_server.py
# 默认监听 http://127.0.0.1:9001
```

服务启动后，根路径可能返回 404，属正常；实际使用 `/create_draft`、`/add_video`、`/save_draft` 等接口。

### 7.5 草稿导入剪映流程

1. 调用 `save_draft` 后，在 `capcut_server.py` 运行目录下生成 `dfd_*` 文件夹  
2. 将整个 `dfd_*` 文件夹复制到剪映草稿目录（见第 3 节）  
3. 重启剪映专业版  
4. 在草稿列表中打开，验证视频轨、字幕、图片是否可编辑

参考：[VectCutAPI README](https://github.com/sun-guannan/VectCutAPI)

## 10. 一键依赖安装（setup-deps）

除第 2.2 节总流程外，可单独执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup-deps.ps1
```

或通过批处理：

```powershell
scripts\setup-deps.bat
```

**脚本会自动完成：**

| 步骤 | 内容 |
|------|------|
| Backend venv | `pip install -r backend/requirements.txt` |
| VectCutAPI 依赖 | `pip install -r vendor/VectCutAPI/requirements.txt`（含 `oss2`） |

| OpenClaw 网关 | `openclaw config patch` → `gateway.auth.mode=none` |

`scripts\start-backend.bat` 每次启动后端前也会调用 `setup-deps`，确保依赖不缺失。

## 13. 各工具在项目中的角色

```text
用户素材 + 备注
        ↓
  OpenClaw Agent（剪辑规划、对话修改）
        ↓
  ASR / VLM（口播转录、画面理解）
        ↓
  unified_timeline.json（唯一事实来源）
        ↓
  ├── HyperFrames → preview.mp4（高级预览）
  └── VectCutAPI  → draft.zip（剪映可编辑草稿）
```

| 工具 | 职责 |
|------|------|
| **OpenClaw** | 调用 LLM、生成 `edit_plan.json` 和 timeline patch、调度工具 |
| **HyperFrames** | HTML 合成 → 无头 Chrome 截帧 → FFmpeg 输出 MP4 |
| **VectCutAPI** | 将时间线元素写入剪映草稿格式（视频/字幕/图片/音频/关键帧） |
| **FFmpeg** | 抽音频、转码、混音 |
| **Chromium** | HyperFrames 渲染引擎的浏览器内核 |
| **faster-whisper** | 本地口播 ASR，输出逐词时间戳 |

## 15. 常见问题

### Q1：`node --version` 报错 `No module named 'nodejs_wheel'`

Python Scripts 里的假 `node.exe` 抢了 PATH。把真实 Node 目录放到 PATH 最前面（见 4.1 节）。

### Q2：OpenClaw 报 Node 版本不足

当前 Node 为 22.14 等低于 22.19 的版本。升级到 22.19+ 后重新运行 `openclaw --version`。

### Q3：剪映里看不到导入的草稿

1. 确认复制到 `com.lveditor.draft` 目录（不是 `Projects` 上一级）  
2. 文件夹名以 `dfd_` 开头  
3. 重启剪映  
4. 确认 `config.json` 中 `is_capcut_env` 与所用软件一致

### Q4：HyperFrames 首次渲染很慢

正常现象。可先输出 720p 预览，或提前下载 Chromium（见 5.2 节）。

### Q5：VectCutAPI 与 HyperFrames 效果不一致

复杂 CSS/GSAP 动效无法完全转为剪映原生元素，需烘焙为视频覆盖层。见需求文档 4.2、9.5 节。

### Q6：口播转录失败 / `No module named 'faster_whisper'`

未创建 `vendor/asr-venv`。执行 `scripts\setup-deps.ps1` 后重启后端。

### Q7：生成在 88% 失败 / `No module named 'oss2'`

VectCutAPI 依赖未装入 backend venv。执行 `scripts\setup-deps.ps1`（会安装 `vendor/VectCutAPI/requirements.txt`）。

### Q8：OpenClaw 对话报 `GatewayCredentialsRequiredError`

执行 `scripts\setup-deps.ps1` 写入 `gateway.auth.mode=none`，并确认已 `npm install -g openclaw`。重启后端后再试。

### Q9：前端 `npm run dev` 报 rolldown 原生模块缺失

用 **Node 22** 重装 `framecraft-agent` 依赖（见 4.1 节）。

### Q10：工作台首页无法上下滚动

已在新版前端修复；拉取最新代码即可。
