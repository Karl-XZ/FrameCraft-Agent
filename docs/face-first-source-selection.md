# 人像优先选源与自动裁切

本项目现在采用“人像优先”选源标准，而不是要求原始视频画面绝对干净。

## 选源标准

- 第一优先级：人物脸部清晰、稳定、无遮挡。
- 第二优先级：人物主体连续说话至少 20 秒，最好 30 到 55 秒。
- 第三优先级：边缘标题条、网页页卡、字幕条、舞台 sponsor 等非脸部污染，可交给项目自动裁切或弱化。
- 不建议使用：
  - 聊天框长期盖住脸部
  - 大面积贴纸长期贴在人物上半身或脸附近
  - 主讲频繁切走、长时间无人物

## 项目行为

- 新版后端不再提供固定 `media/timeline_builder/hyperframes_service` 流水线。
- 当前项目的单一 Agent 必须在分析素材时抽帧或探测画面，判断人物脸部位置和可裁切区域。
- Agent 写 timeline 与 HyperFrames 工程时必须把人脸安全区作为硬约束，优先裁掉边缘污染，不能裁掉脸部、嘴部或关键表情。
- 视觉复审若发现人物脸部、嘴部或字幕被遮挡，必须重做设计并重新渲染。

## 可复现批量流程

- 配置文件可写 `clip_start` / `clip_end`，先从长视频截出连续人像片段，再把该片段作为“完整源片”上传给项目。
- 批处理脚本：
  - `scripts/run_online_codex_batch.py`
- 推荐配置样例：
  - `configs/online_talking_heads_facefirst_20260618.json`
